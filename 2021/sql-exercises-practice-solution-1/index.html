MySQL习题总结

1、输出第二高的薪水，若没有第二高，返回NULL

解法一：使用两次聚合函数，先找到薪资最大的并去掉，再查找剩下的中最大的。

select max(Salary) SecondHighestSalary from Employee WHERE Salary <> (Select max(Salary) FROM Employee);

缺点是相当于两次排序，如果取第N高，这种思想无法解题。且若无第二高，无法返回NULL。

解法二：将不同的薪资按降序排序，然后使用 LIMIT 子句获得第二高的薪资。

select distinct(Salary) SecondHighestSalary from Employee order by Salary desc limit 1 offset 1;

缺点是如果没有第二高工资，这个解决方案将被判断为 “错误答案”，因为本表可能只有一项记录。因此可以将结果作为临时表。

select (select distinct(Salary) from Employee order by Salary desc limit 1 offset 1) as SecondHighestSalary;

或

select ifnull((select distinct(Salary) from Employee order by Salary desc limit 1 offset 1), null) as SecondHighestSalary;

1）本题考点：LIMIT OFFSET分页、IFNULL使用、DISTINCT去重使用

SELECT * FROM EMP LIMIT 9, 4;

SELECT * FROM EMP LIMIT 4 OFFSET 9;

上述两条语句都显示第10-13行数据，前者语句为从第10行开始，显示后4行，后者语句为显示4行数据，从第9行开始。或者将OFFSET理解为跳过9条记录，也即从第10行开始，这样理解上述两语句等价。

AS 表示用作、当成，作为，一般是重命名列名或者表名，可省略。

2）拓展：返回第N高数据

select ifnull((select distinct(Salary) from Employee order by Salary desc limit 1 offset N-1), null) as SecondHighestSalary;

CREATE FUNCTION getNthHighestSalary(N INT) RETURNS INT
BEGIN
set N=N-1;
RETURN (
# Write your MySQL query statement below.
select distinct Salary from Employee order by Salary desc limit 1 offset N
);
END

注意函数中的语句不能解决NULL值，且RETURN中的语句不写结束符号”;"，且LIMIT语句后不能使用运算符，LIMIT 1 OFFSET N-1会报错，因此需使用SET语句提前赋值。

SET语句可用于向系统变量或用户变量赋值。

MySQL中用户变量以”@“开始，形式为”@var_name”，用以区分用户变量及列名。但在函数内部，变量不需要加”@"。

SET语句可以使用=或:=来赋值，SELECT语句只能使用:=来赋值。

2、编写一个 SQL 查询来实现分数排名。

select Score, (dense_rank() over(order by Score desc)) as `Rank` from Scores order by Score desc;

注意MySQL中有rank函数，所以新列名需要转化为转义字符。

1）本题考点：排序

DENSE_RANK()用法为

DENSE_RANK() OVER(PARTITION BY 划分列名 ORDER BY 排序列名) 新列名

PARTITION BY表示按照划分列将生成的结果集划分分区，ORDER BY表示在每个分区中按排序列进行排序，OVER表示将DENSE_RANK函数应用在每个分区中。本语句可以放在SELECT语句的查询列中。

MySQL中一共有两个排序函数，现给定五个成绩99，98，85，80，75，则DENSE_RANK()返回1，1，2，3，4，排名不间断；RANK()返回1，1，3，4，5，排名间断。ROW_NUMBER()会返回1，2，3，4，5。

定义变量为可变长度的字符串时必须给出varchar的最大长度，比如varchar(10)。

3、编写一个 SQL 查询，查找所有至少连续出现三次的数字。

首先观察Logs表

id num
1 1
2 1
3 1
4 2
5 1
6 2
7 2

如果用row_number()函数用num作为分区，按id从小到大排序，可得如下结果

select id, num, row_number() over(partition by num order by id) a from myLogs;
id num a
1 1 1
2 1 2
3 1 3
5 1 4
4 2 1
6 2 2
7 2 3

如果num是连续的，则id列与a列相减后得到的数值是一样的

select id, num, id-row_number() over(partition by num order by id) a from myLogs;
id num a
1 1 0
2 1 0
3 1 0
5 1 1
4 2 3
6 2 4
7 2 4

按num和a分组，并统计个数，筛选出个数>=3的num

select num, count(*) from (select id, num, id-row_number() over(partition by num order by id) a from myLogs) b group by num, a having count(*)>=3;
num count(*)
1 3

把上述结果作为临时表，将num返回并命名为ConsecutiveNums

select distinct num as ConsecutiveNums from (select num, count(*) from (select id, num, id-row_number() over(partition by num order by id) a from myLogs) b group by num, a having count(*)>=3) c;
ConsecutiveNums
1

1）本题另外解法：

首先观察三表连接后，把三个相邻的结果排列在一行

select * from myLogs l1, myLogs l2, myLogs l3 where l3.id-1=l2.id and l2.id-1=l1.id;
id num id num id num
1 1 2 1 3 1
2 1 3 1 4 2
3 1 4 2 5 1
4 2 5 1 6 2
5 1 6 2 7 2

挑选出那些所有num都相等的行，则该行对应的num值就是连续出现三次的数字

select l1.num as ConsecutiveNums from myLogs l1, myLogs l2, myLogs l3 where l3.id-1=l2.id and l2.id-1=l1.id and l1.num=l2.num and l2.num=l3.num;

思路很好，利用id约束将相邻的两条记录放在一行，并进行后续相邻两条数据间的计算。缺点，如果是查询连续出现N次的数字，本语句过于繁琐和耗时。

4、查找超过经理收入的员工

Employee表包含所有员工，他们的经理也属于员工。每个员工都有一个 Id，此外还有一列对应员工的经理的 Id。编写一个 SQL 查询，该查询可以获取收入超过他们经理的员工的姓名。在上面的表格中，Joe 是唯一一个收入超过他的经理的员工。

Id Name Salary ManagerId
1 Joe 70000 3
2 Henry 80000 4
3 Sam 60000 NULL
4 Max 90000 NULL 
select e1.`Name` as `Employee` from Employee e1 inner join Employee e2 on e1.ManagerId=e2.Id and e1.Salary>e2.Salary

1）本题考点：INNER JOIN

回顾三种连接的区别，若为LEFT JOIN，则e1表中第1和2条记录一定会被匹配后出现，e1中的第3和4条由于ManagerId为NULL，无法匹配，但由于是左连接，必须出现，因此将e1中的3、4记录后面补NULL后保留；若为RIGHT JOIN，则e2表中第1和2条记录一定会被匹配后出现，e2的第3和4条由于ManagerId为NULL，但由于是右连接，必须出现，因此将e2中的3、4记录前面补NULL后保留；若为INNER JOIN，内连接，不符合匹配的都不出现，因此只有第1、2条记录。

5、 查找重复的电子邮箱

编写一个 SQL 查询，查找 Person 表中所有重复的电子邮箱。

Id Email
1 a@b.com
2 c@d.com
3 a@b.com

应返回

Email
a@b.com

如下解法对HAVING的使用不熟练

select p.Email from (select Email, count(*) from Person group by Email having count(*)>1) p;

注意，若不将FROM后的表另外命名，前面SLEECT Email会报错，‘ERROR 1248 (42000): Every derived table must have its own alias’。

HAVING可以如下使用

select Email from testPerson group by Email having count(*)>1;

即分组列必须出现，但聚合函数可以不出现在SELECT中。本题返回的是分组列，因此刚好可以在SELECT中隐去聚合函数。

1）本题扩展：如何查找重复数据？

看到“找重复”的关键字眼，首先要用分组函数（group by），再用聚合函数中的计数函数count()给分组列计数。分组汇总后，加上条件选出计数大于1的分组列，就是重复的数据。
