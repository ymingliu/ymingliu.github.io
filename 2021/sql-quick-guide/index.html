MySQL Quick Guide

MySQL速查手册

DDL

查看数据库

SHOW DATABASES;

切换数据库

USE 数据库名;

创建数据库

CREATE DATABASE 数据库名;

删除数据库

DROP DATABASE 数据库名;

查看数据库中的表

SHOW TABLES;

创建表

CREATE TABLE 表名(
列名1 列类型 PRIMARY KEY AUTO_INCREMENT,
列名2 <INT(a) | DOUBLE | DECIMAL(a, b) | CHAR(a) | VARCHAR(a)> UNSIGNED NOT NULL UNIQUE DEFAULT 默认值,
...
)COMENT

查看表结构

DESC 表名;

修改表之添加列

ALTER TABLE 表名 ADD(
列名1 列类型,
列名2 列类型,
...
);

修改表之修改列类型（一次只能修改一列，不能同时修改多列）

ALTER TABLE 表名 MODIFY(列名, 列类型);

修改表之删除列（一次只能删除一列，不能同时删除多列）

ALTER TABLE 表名 DROP 列名;

修改表之表名的重命名

ALTER TABLE 表名 RENAME [TO] 新表名;

修改表之列名的重命名

ALTER TABLE 表名 CHANGE 旧列名 新列名 新列类型;

如果旧列名和新列名相同，则可实现只修改了列类型，相当于MODIFY

ALTER小结

ALTER TABLE 表名
ADD --> 添加列
MODIFY --> 修改列名和列类型
CHANGE --> 修改列名
DROP --> 删除列
RENAME [TO] --> 修改表名
ADD PRIMARY KEY(列名) --> 为列添加主键
DROP PRIMARY KEY  --> 删除主键
DROP FOREIGN KEY  --> 删除外键

DML

查询表记录（DQL）

SELECT * FROM 表名;

插入表记录

INSERT INTO 表名(
列名1, 列名2, ...
)VALUES(
列1值, 列2值, ...
)
  • 若存在自增的主键列，插入第一条记录时该列可以不用给值，若只是主键列，是要给的，否则会违背主键约束的三个性质之一，非空。
  • 可以指定所有列、指定部分列与不指定列，对于未指定的列，赋值NULL

修改表记录

UPDATE 表名 SET 列名=值 WHERE 修改条件

修改表记录之修改条件

  • OR 或
  • AND 且
  • BETWEEN … AND … 值在两者之间，多用于数值型
  • IN() 值在内部，多用于字符型
  • IS [NOT] NULL 是否为空，注意任何值=NULL都返回FALSE

删除表记录

DELETE FROM 表名 WHERE 删除条件;
  • 不加WHERE条件，表记录会全部删除
  • 删除表记录是DELETE，删除表是DROP

DQL

查询时去掉重复列

SELECT DISTINCT 列名 FROM 表名;

对NULL值转换

IFNULL(列名, 转换值)

连接字符串

CONCAT(列名, 列名)

模糊查询

LIKE 语句，'_'匹配一个字符，'%'匹配0个或多个字符

排序

ORDER BY 列名 <ASC | DESC> 默认ASC升序

聚合函数

  • COUNT 计数
  • SUM 求和
  • MAX 最大
  • MIN 最小
  • AVG 平均

分组

GROUP BY 分组列必须出现在查询列中

HAVING 后是分组后的条件，条件内容都是聚合函数，也即若查询语句中涉及聚合函数条件，一定是放在GROUP BY后的HAVING中，而不是WHERE后。

DQL语句执行顺序

SELECT，FROM，WHERE，GROUP BY，HAVING，ORDER BY，LIMIT

  • 注意MySQL的机读顺序FROM，WHERE，GROUP BY，HAVING，SELECT，ORDER BY，LIMIT

显示部分查询结果

LIMIT a 从第1条至第a条

LIMIT a, b 从第a+1条至第a+b条

  • 若一页记录m行，查询第n页，则为LIMIT m*(n-1), m;

其他

主键约束

  • 创建表时指定主键
CREATE TABLE 表名(
列名 列类型 PRIMARY KEY,
...
)

或者

CREATE TABLE 表名(
列名 列类型,
...
PRIMARY KEY(列名)
)
  • 表已存在时添加主键
ALTER TABLE 表名 ADD PRIMARY KEY(列名);
  • 删除主键
ALTER TABLE 表名 DROP PRIMARY KEY;

主键自增长

AUTO_INCREMENT

  • 由于唯一、非空的特点，通常将主键列定义为整型，并设置自增长，不用再手动插入。
  • 表格内容删除后，再插入新内容时主键依旧保持自增长。
  • 设置了NOT NULL和UNIQUE的列依然不是主键列，因为不能被外键引用

外键约束

一张表有一个主键，同时也可能包含一个外键，它是另一张表（或者本表）的主键

  • 外键列必须引用主键、可以重复、可以为空、一张表可以有多个外键
  • 主要用于防止无效数据插入外键列，因为其值必须是它引用的主键列的值之一。

创建表时添加外键列

CONSTRAINT 外键约束名
FOREIGN KEY 外键名 (主键列1, 主键列2,...)
REFERENCES 外键所依赖的表(外键列1, 外键列2,...)
ON DELETE <SET NULL|NO ACTION|RESTRICT>
ON UPDATE <SET NULL|NO ACTION|RESTRICT>
  • ON DELETE/UPDATE 设置级联删除/更新操作，以便父表信息删除时子表也进行相关操作。
  • SET NULL 将子表外键列值设置为NULL，NO ACTION和RESTRICT表示拒绝删除子表相关联的数据。
  • 如不命名 FOREIGN KEY 约束，则’CONSTRAINT 外键约束名’一同省略。

为已有表添加外键列

ALTER TABLE 表名 ADD
CONSTRAINT 外键约束名
FOREIGN KEY 外键名 (主键列1, 主键列2,...)
REFERENCES 外键所依赖的表(外键列1, 外键列2,...)
ON DELETE <SET NULL|NO ACTION|RESTRICT>
ON UPDATE <SET NULL|NO ACTION|RESTRICT>

多表查询之连接查询

内连接、左[外]连接和右[外]连接

表1 INNER JOIN 表2 ON 连接条件

表1 RIGHT [OUTER] JOIN 表2 ON 连接条件

ALL/ANY 返回符合所有/任意结果的记录。

索引

创建索引

CREATE INDEX 索引名 ON 表名(列名1, 列名2,…);

添加索引

# 为某列（或多列）添加一个主键（或联合主键），这时的主键可以又作为索引，由主键的特性可知，该索引值是唯一的且不能为NULL
ALTER TABLE tableName ADD PRIMARY KEY(columnName_list)；
# 该语句创建的索引是唯一的，但可以有NULL，且NULL可以有多个
ALTER TABLE tableName ADD UNIQUE indexName(columnName_list);
# 添加普通索引，索引值不唯一且可为NULL
ALTER TABLE tableName ADD indexName(columnName_list);
# 指定添加的索引为FULLTEXT，用于全文索引
ALTER TABLE tableName ADD FULLTEXT indexName(columnName_list);

删除索引

DROP INDEX ON 表名;

查看索引

SHOW INDEX FROM 表名;
